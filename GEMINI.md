# GEMINI.md

이 파일은 Gemini Code(또는 다른 AI 에이전트)가 이 저장소에서 작업할 때 참고해야 할 컨텍스트 가이드입니다. 프로젝트 구조, 워크플로우 및 준수해야 할 엄격한 규칙들을 정의합니다.


---


## 프로젝트 개요

이 저장소는 게임 개발, 특히 **언리얼 엔진 5(UE5)**, **VFX(Niagara)**, **최적화**에 초점을 맞춘 **지식 베이스**입니다. 여러 개의 독립적인 **Obsidian 볼트(Vault)**로 구성되어 있으며, 주로 **한국어**로 작성되었습니다.


---


## 디렉토리 구조 및 아키텍처

저장소는 **멀티 볼트 아키텍처**를 따릅니다. 각 최상위 디렉토리는 독립적인 Obsidian 볼트로 기능합니다.

```text
D:\Obsidian\
├── Unreal/          # [Vault] UE5 기초, 아키텍처, 게임플레이 프레임워크
├── Niagara/         # [Vault] VFX 시스템, 이펙트 제작 및 최적화
├── Optimization/    # [Vault] 성능 프로파일링, 메모리 관리, 카오스 피직스
├── Tools/           # [Vault] 개발 도구, 워크플로우, 스크립팅
├── 3DCG_Channel/    # [Vault] 3D 모델링 노트 (Max, Maya, Blender, ZBrush, Substance)
├── Private/         # [Private] 개인 노트 (Git 제외)
├── Script/          # 헬퍼 스크립트 (Python 등)
├── Web/             # 웹 관련 리소스 또는 변환 결과물
└── README.md        # 프로젝트 소개
```

## 운영 가이드라인

**파일을 생성하거나 수정할 때 다음 규칙을 엄격히 준수하십시오.**


### 0. 언어 설정

*   **AI 응답**: 사용자와의 모든 대화 및 응답은 **항상 한국어**로 수행합니다.
*   **콘텐츠**: 모든 마크다운 문서의 본문은 **한국어**로 작성합니다.
    *   명확성을 위해 필요한 경우 기술 용어는 영어를 병기하거나 유지하되, 설명 문장은 반드시 한국어여야 합니다.


### 1. 에셋 관리

*   **위치**: 모든 첨부 파일(이미지, GIF, MP4, PDF)은 작업 중인 **특정 볼트 루트**의 `Resources/` 폴더에 저장해야 합니다.
    *   *예*: `Unreal/Resources/`, `Optimization/Resources/` 등.
*   **삽입**: 표준 Obsidian 문법을 사용합니다: `![[파일명.확장자]]`.
*   **이름**: 에셋 파일명은 내용을 설명하는 영어 이름을 사용합니다.


### 2. 마크다운 및 콘텐츠 구조

*   **형식**: Obsidian 위키링크(`[[링크]]`)를 포함한 표준 마크다운을 사용합니다.
*   **파일 배치**:
    *   새 노트는 볼트 내의 적절한 하위 디렉토리에 배치합니다.
    *   노트 파일명은 내용을 설명하는 한국어 이름을 사용합니다.
    *   `Init.md` 파일은 볼트 진입점 역할을 하므로 임의로 삭제하지 마십시오.


### 3. 문서 편집 및 스타일 가이드

문서의 **가독성**을 최우선으로 하여 다음 규칙을 적용합니다.

*   **여백 및 구조 (Spacing)**:
    *   **H2 영역**: 위아래 구분선(`---`)을 포함하여 총 3줄의 공백을 확보합니다.
    *   **H3 영역**: 본문과 구분선 없이 위쪽으로 2줄의 공백을 확보합니다.
    *   **H4 영역**: 본문과 구분선 없이 위쪽으로 2줄의 공백을 확보합니다.
*   **텍스트 스타일링 (Styling)**:
    *   **볼드(`**...**`)**: 핵심 개념, 중요 강조 사항, UI 경로(예: `File -> Open`).
    *   **인라인 코드(` `...` `)**: 파일명, 경로, 클래스명, 함수명, 변수값, 기술 용어.
*   **원문 수정 원칙**:
    *   단순 오타나 문법 오류는 즉시 수정합니다.
    *   내용의 의미를 변경하는 원문 수정은 **금지**하며, 불가피한 경우 사용자에게 보고 후 승인을 받습니다.


### 4. 작업 프로세스 (Workflow)

효율적이고 안전한 작업을 위해 다음 순서를 따릅니다.

1.  **분석 (Understand)**:
    *   작업할 볼트의 위치와 기존 파일 구조를 파악합니다.
    *   `GEMINI.md`와 `CLAUDE.md`의 규칙을 상기합니다.
2.  **실행 (Implement)**:
    *   내용 작성 시 **스타일 가이드(볼드, 인라인 코드, 공백)**를 즉시 적용합니다.
    *   불필요한 수정 단계를 줄이기 위해, 초안 작성 시 가독성을 고려합니다.
3.  **검증 (Verify)**:
    *   마크다운 문법 오류를 확인합니다.
    *   에셋 링크(`![[...]]`)가 올바른지 확인합니다.
4.  **커밋 (Commit)**:
    *   아래 컨벤션을 준수하여 커밋합니다.


### 5. Git 컨벤션

*   **커밋 메시지 접두사**:
    *   `[NEW] <파일명 또는 설명>`: 새로운 콘텐츠 추가.
    *   `[FIX] <파일명 또는 설명>`: 기존 콘텐츠 수정 및 교정.
*   **제외 파일**:
    *   `Private/` 폴더 내 파일.
    *   `.obsidian/` 내의 설정 파일(`workspace.json`, `app.json`).
